RESURSE MINIME DE OPERARE
peviitor.ro trebuie să poată funcționa cu resurse minime. Proiectul este construit să reziste și să rămână accesibil chiar și atunci când bugetele sunt zero, iar infrastructura este minimalistă.

Principiu

Un proiect construit de voluntari nu poate depinde de resurse costisitoare. Fiecare componentă este aleasă sau construită astfel încât să poată rula pe un hardware modest, cu costuri lunare aproape de zero și cu întreținere minimă.

Resurse umane minime

  • O persoană — proiectul poate fi întreținut de o singură persoană dedicată care gestionează serverul, aprobă PR-uri și răspunde pe Discord
  • Fără dependență unică — documentația și codul sunt scrise astfel încât orice alt voluntar poate prelua rapid
  • Self-service — scraperele pot fi propuse și aprobate fără intervenția administratorului principal, printr-un proces semi-automat

Cerințe hardware pentru SOLR

Apache SOLR rulează pe un cluster de Raspberry Pi în rețeaua AOC. Configurarea actuală:

  • SOLR — Raspberry Pi 5, 16 GB RAM, heap SOLR 8 GB, SSD NVMe extern
  • API — Raspberry Pi 5, 4 GB RAM, server backend
  • Test — Raspberry Pi 4, 2 GB RAM, mediu izolat de testare
  • Backup — Raspberry Pi 400, 4 GB RAM, preia orice rol în caz de cădere
  • Rețea — conexiune stabilă la internet, bandwidth minim 10 Mbps

Cerințe software minime

  • Ubuntu Server LTS — sau orice Linux stabil
  • Docker Engine — pentru SOLR containerizat
  • Docker Compose — pentru orchestrare locală
  • Git — pentru controlul versiunilor
  • OpenJDK 11+ — necesar pentru SOLR
  • Nginx — sau orice reverse proxy

Toleranță la căderi

În cazul în care serverul SOLR cade (ex. pană de curent, problemă hardware), proiectul nu moare:

  • Backup zilnic — indexul SOLR este exportat și stocat în GitHub Releases
  • Restaurare rapidă — un nou server poate fi pornit în < 30 de minute folosind ultimul backup
  • Front-end static — site-ul rămâne accesibil pe GitHub Pages chiar dacă serverul SOLR este jos (cu date ușor învechite)

Feature-uri derivate

  • Minimal Deployment Guide — ghid de deploy pe Raspberry Pi
  • Docker Compose One-liner — pornire întreg stack cu o singură comandă

  • Health Check Endpoint — endpoint pentru monitorizare
  • Resource Monitoring Dashboard — panel cu utilizare CPU, RAM, disk
  • Disaster Recovery Plan — plan pas-cu-pas pentru recuperare după cădere
