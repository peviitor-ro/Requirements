ID: EPIC-007 Epic
API PUBLIC

Prezentare generală

peviitor.ro pune la dispoziție un API public prin care orice companie sau scraper poate publica, actualiza și șterge automat anunțuri de angajare. API-ul servește și frontend-ul principal ca sursă de date pentru căutări.

Acest epic definește arhitectura API-ului, flow-ul de publicare a unui job cu identificarea și validarea automată a companiei, endpoint-urile disponibile și regulile de securitate.

Roluri API

API-ul are trei roluri distincte:

Rol Scop Consumator
BFF Servește date pentru interfața web — căutare, filtrare, paginare Frontend peviitor.ro
Scraper Primește date de la scrapere automate — push job-uri, update status GitHub Actions, scrapere voluntari
Company Oferă acces direct companiilor să își gestioneze propriile anunțuri Companii partenere, HR, ATS

Flow publicare job (Company API)

Când o companie trimite un anunț, API-ul execută un lanț de validări înainte de a-l salva în SOLR:

  1. Extragere identitate companie — API-ul identifică compania din CIF, denumire sau locație:
    • Dacă există cif → lookup în SOLR și ANAF
    • Dacă există doar company → căutare ANAF după nume + locație
    • Dacă nu există nici CIF nici company name → REJECT (red-flag)
  2. Validare companie — se interoghează ANAF API cu CIF-ul identificat:
    • Confirmare că denumirea din ANAF corespunde cu cea trimisă
    • Dacă ANAF returnează denumire diferită → corectăm automat
    • Dacă ANAF nu găsește compania → REJECT (companie fictivă)
  3. Îmbogățire date — se completează câmpurile companiei din surse oficiale:
    • company, cif, status, location din ANAF
    • brand, group, website, career din SOLR sau AI web search
  4. Validare job — se verifică câmpurile anunțului conform modelului:
    • url → obligatoriu, valid, accesibil, unic
    • title → obligatoriu, max 200 caractere, fără cuvinte interzise
    • cif → obligatoriu, trebuie să corespundă cu Company.id
    • company → obligatoriu, trebuie să corespundă cu Company.company
    • Verificare că job-ul nu există deja (după URL)
  5. Insert în SOLR — documentul complet, cu toate câmpurile populate, este salvat în index.

Reguli de reject (red-flag)

Următoarele situații duc la respingerea automată a anunțului:

Situație Motiv
Fără CIF și fără company name Nu știm cine publică anunțul
CIF invalid (nu există în ANAF) Companie fictivă
Titlu suspect (ex: "Ionel angajează") Conținut neadecvat, nu este un anunț real
Cuvinte urate sau mesaje mascate în titlu Încălcarea regulilor de conținut
URL către site malițios sau obscen Pericol pentru securitatea utilizatorilor
Companie cu status inactiv sau radiat Compania nu mai există legal
Job duplicat (același URL există deja) Evităm dublurile în baza de date

Endpoint-uri

Metoda Path Rol Descriere
GET /api/v1/search BFF Căutare full-text job-uri cu filtrare și paginare
POST /api/v1/job Scraper / Company Publicare job nou cu validare și îmbogățire
PUT /api/v1/job/{url} Scraper / Company Actualizare job existent
DELETE /api/v1/job/{url} Scraper / Company Ștergere job (doar ale proprii)
GET /api/v1/company/{cif} BFF / Company Detalii companie
POST /api/v1/auth/register Company Înregistrare companie și obținere cheie API

Autentificare și securitate

  • API Key — fiecare companie primește o cheie API unică la înregistrare, trimisă în header-ul Authorization: Bearer <key>.
  • Autorizare per companie — un API key poate opera doar asupra job-urilor companiei respective.
  • Rate limiting — max 100 request-uri per minut per companie, 1000 per minut pentru scrapere.
  • Audit log — toate operațiile de scriere sunt înregistrate cu timestamp, identitate și payload.
  • HTTPS obligatoriu — orice request trebuie să vină pe HTTPS.

Documentație

API-ul trebuie să aibă documentație OpenAPI/Swagger accesibilă public la /api/v1/docs.

  • Endpoint-uri — listă completă cu request/response exemple
  • Autentificare — cum se obține și folosește un API key
  • Modele — scheme JSON pentru Job și Company
  • Erori — coduri de eroare și semnificația lor
  • Ghid companii — cum se integrează o companie în 5 minute

Feature-uri derivate

  • Company Identity Resolution — identificarea companiei din CIF, nume și locație
  • ANAF Enrichment Pipeline — completare date companie din ANAF
  • Job Validation Pipeline — lanțul de validări înainte de insert
  • Red-Flag Detection — detectare pattern-uri suspecte în titluri și URL-uri
  • Company Self-Service — înregistrare și gestionare cont companie
  • API Documentation — documentație OpenAPI și ghiduri
  • Rate Limiting — protecție împotriva abuzului
  • Audit Log — înregistrare operații de scriere
