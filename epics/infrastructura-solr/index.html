ID: EPIC-006 Epic
INFRASTRUCTURĂ SOLR

Prezentare generală

În inima peviitor.ro se află Apache SOLR, motorul de căutare full-text care stochează și indexează toate job-urile și companiile. Sistemul are două core-uri principale: job și company.

Acest epic definește schemele SOLR, configurarea core-urilor, indexarea, căutarea full-text și optimizarea performanței.

Căutare full-text

  • Câmpuri indexate — următoarele câmpuri sunt copiate în câmpul _text_ pentru căutare full-text:
    • Job: url, title, company, location, tags, workmode, salary
    • Company: company, location, website, brand, group
  • Analizor românesc — text_general folosește un analizor care păstrează diacriticele, astfel încât căutările cu diacritice să funcționeze corect.
  • Auto-complete — un SugestComponent pe câmpul title oferă sugestii de autocompletare în interfață.

Configurare SOLR

  • Schema API — câmpurile sunt adăugate via SOLR Schema API în scripturi de configurare (shell pentru Ubuntu/Pi, batch pentru Windows).
  • Autentificare — SOLR este securizat cu utilizator și parolă configurate în opencode.json.
  • Docker — SOLR rulează în container Docker, cu volume persistente pentru date.

Cluster și redundanță

  • SOLR Cloud — pentru producție, se recomandă configurarea SOLR în mod cloud cu cel puțin 2 noduri.
  • Replicare — fiecare core are cel puțin o replică pe un nod diferit.
  • Sharding — la volume mari de date (peste 10M documente), indexul poate fi partiționat pe shard-uri.

Feature-uri derivate

  • SOLR Schema Setup — scripturi de configurare a schemelor job și company
  • Full-Text Search — configurarea căutării full-text cu analizor românesc
  • Autocomplete Suggester — configurarea sugestiilor de autocompletare
  • SOLR Cloud Cluster — configurarea clusterului SOLR cu replicare
  • SOLR Backup & Restore — backup și restore pentru core-uri
  • Performance Tuning — optimizare cache, thread pool, query time
