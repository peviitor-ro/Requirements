ASOCIAȚIA OPORTUNITĂȚI ȘI CARIERE
ID: EPIC-001 Epic
DATE DESPRE COMPANII

Prezentare generală

Datele despre companii stau la baza întregului ecosistem peviitor.ro. Fiecare loc de muncă publicat aparține unei companii, iar acuratețea informațiilor despre acestea determină calitatea agregării.

Acest epic definește modelul de date al companiilor, sursele din care datele sunt colectate și menținute, precum și ciclul de viață al unei companii în sistem.

Modelul companiei

Definiția câmpurilor care descriu o companie în cadrul sistemului:

Câmp Tip Obligatoriu Descriere
id string Da CIF/CUI, exact 8 cifre, fără prefix RO
company string Da Denumirea legală din Registrul Comerțului, uppercase, cu diacritice
brand string Nu Numele comercial sub care este cunoscută compania (ex: ORANGE, EPAM)
group string Nu Grupul din care face parte compania (ex: Orange Group, EPAM Systems)
status string Nu activ / suspendat / inactiv / radiat
location string[] Nu Orașul/orașele unde compania are sedii, diacritice acceptate
website string[] Nu URL-urile site-urilor oficiale
career string[] Nu URL-urile paginilor de carieră
lastScraped string Nu Data ultimei colectări, format ISO8601
scraperFile string Nu Numele fișierului scraper care a adăugat compania

Exemplu: Holcim România

Câmp Valoare
id 361324
company HOLCIM ROMÂNIA S.A.
brand HOLCIM
group Holcim Group
status activ
location Arad, Brașov, București, Cluj-Napoca, Constanța, Iași, Timișoara
website https://www.holcim.ro
career https://www.holcim.ro/cariere
lastScraped 2026-06-26
scraperFile holcim.md

Surse de date

Datele despre companii provin din următoarele surse:

1. ANAF API

API-ul pus la dispoziție de Agenția Națională de Administrare Fiscală. Oferă denumirea oficială, starea de înregistrare și adresa sediului social.

  • Endpoint: https://webservicesp.anaf.ro/api/PlatitorTvaRest/v9/tva
  • Limită: 100 CUI-uri per request, maxim 1 request/secundă

2. ONRC

Registrul Comerțului pune la dispoziție seturi de date deschise pe data.gov.ro. Acestea conțin toate companiile înregistrate în România, în format CSV.

3. Scrapere directe

Voluntarii adaugă companii manual prin scraper-e. Denumirea companiei trebuie să corespundă cu cea din Registrul Comerțului.

4. AI agentic (Web Search)

Un agent AI cu funcție de căutare web identifică automat datele lipsă ale companiei: website oficial, pagină de carieră, brand și grupul din care face parte.

  • Date descoperite: website, career, brand, group
  • Metodă: interogare web search cu numele companiei și locația
  • Avantaj: elimină munca manuală de documentare pentru fiecare companie

Ciclul de viață

Ciclul de viață al unei companii în sistem:

  1. Adăugare — Un voluntar creează un scraper care menționează o companie. Compania este adăugată în SOLR cu datele minime (id, company, status=activ).
  2. Îmbogățire — Un proces automat completează datele lipsă: status oficial, locație, website, pagină de carieră, brand, grup.
  3. Verificare periodică — Companiile active sunt re-verificate împotriva surselor oficiale pentru a detecta companiile radiate sau inactive.
  4. Arhivare — Companiile cu status inactiv sau radiat nu mai au job-uri active asociate.

Feature-uri derivate

Feature-urile derivate din acest epic, ce vor fi detaliate separat:

  • ANAF Sync — sincronizare automată cu ANAF
  • ONRC Import — import date din Registrul Comerțului
  • Company Validation — reguli de validare a datelor de companie
  • Brand Discovery — identificare brand comercial
  • Group Discovery — identificare grup din care face parte
  • Automation & Scheduling — GitHub Actions pentru actualizări periodice
