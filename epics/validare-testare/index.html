ID: EPIC-005 Epic
VALIDARE & TESTARE

Prezentare generală

Fiecare loc de muncă din peviitor.ro trece printr-un lanț de validări înainte de a fi publicat. Regulile de validare asigură că datele sunt corecte, complete și sigure pentru utilizatori.

Acest epic definește cele 12 reguli de validare a modelului Job, arhitectura testelor și modul de raportare a erorilor.

Reguli de validare

Fiecare câmp al modelului Job are propria regulă de validare:

# Câmp Severitate Regulă
TC01urlERRORObligatoriu, URL valid HTTP/HTTPS
TC02titleERRORObligatoriu, max 200 caractere, fără HTML, fără spații la capete
TC03companyWARNDacă există, trebuie să fie string ne-gol și să corespundă cu Company.company
TC04cifWARNDacă există, trebuie să fie string ne-gol și să corespundă cu Company.id
TC05locationWARNDacă există, trebuie să fie string sau array de stringuri
TC06tagsWARNArray, max 20 de elemente, lowercase, fără diacritice
TC07workmodeWARNUna dintre: remote, on-site, hybrid
TC08dateWARNFormat ISO8601 UTC valid
TC09statusWARNUna dintre: scraped, tested, published, verified
TC10vdateWARNISO8601 valid, setat doar când status = verified
TC11expirationdateWARNISO8601, trebuie să fie ≤ vdate + 30 zile
TC12salaryWARNString, format MIN-MAX CURRENCY sau VALOARE CURRENCY

Severitate

  • ERROR — câmpul este obligatoriu. Dacă lipsește sau este invalid, anunțul este respins și nu ajunge în baza de date.
  • WARN — câmpul este opțional. Dacă există și este invalid, se generează un avertisment, dar anunțul poate fi salvat cu o notă.

Arhitectura testelor

Testele sunt scrise în PHP și rulează automat:

  • Teste unitare — fiecare regulă de validare (TC01–TC12) are propriul fișier de test.
  • Teste de integrare — un test complet care rulează toate validările pe un set de date de test.
  • Runner — un script central care încarcă toate testele, le rulează și agregă rezultatele.
  • Raportare — rezultatul este un raport JSON/HTML cu: test trecut/căzut, erori, avertismente.

Integrare în pipeline

  • GitHub Actions — testele rulează automat la fiecare push și pull request.
  • Blocant — dacă un test ERROR cade, pipeline-ul eșuează și schimbarea nu este aprobată.
  • Raport — rezultatele testelor sunt publicate ca artifact și vizibile în GitHub.
  • Alerts — testele căzute în producție generează alertă pe Discord.

Configurare

Parametrii configurabili ai validării:

  • MAX_EXPIRATION_DAYS — 30 zile (perioada maximă de valabilitate a unui job)
  • MAX_TAGS — 20 (număr maxim de tag-uri per job)
  • MAX_TITLE_LENGTH — 200 caractere
  • WORK_MODES — remote, on-site, hybrid
  • JOB_STATUSES — scraped, tested, published, verified

Feature-uri derivate

  • Validation Engine — motorul de validare care rulează toate regulile
  • Test Runner — scriptul central de rulare a testelor
  • Test Report Generator — generator rapoarte JSON/HTML
  • GitHub Actions Test Pipeline — workflow-ul de testare
  • Validation Config — configurarea parametrilor de validare
