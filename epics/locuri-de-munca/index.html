ASOCIAȚIA OPORTUNITĂȚI ȘI CARIERE
ID: EPIC-002 Epic
LOCURI DE MUNCĂ

Prezentare generală

Misiunea peviitor.ro este de a agrega toate locurile de muncă din România într-un singur loc. Fiecare anunț trebuie să aibă o structură clară, date corecte și să fie ușor de găsit de către cei care își caută un loc de muncă.

Acest epic definește modelul de date al unui loc de muncă, sursele din care sunt colectate anunțurile, ciclul de viață al unui anunț în sistem și regulile de validare.

Modelul locului de muncă

Definiția câmpurilor care descriu un loc de muncă în cadrul sistemului:

Câmp Tip Obligatoriu Descriere
url string Da URL-ul unic al paginii de job. Cheie unică în sistem.
title string Da Titlul exact al poziției, max 200 caractere, fără HTML, diacritice acceptate
company string Da Numele companiei care angajează, uppercase, trebuie să corespundă cu Company.company
cif string Da CIF/CUI al companiei, trebuie să corespundă cu Company.id
location string[] Nu Orașul/orașele unde se desfășoară activitatea, diacritice acceptate
tags string[] Nu Skill-uri, educație, experiență, max 20 de intrări, lowercase, FĂRĂ diacritice
workmode string Nu remote / on-site / hybrid
salary string Nu Salariu în format MIN-MAX CURRENCY (ex: 5000-8000 RON)
date date Nu Data colectării/indexării, format ISO8601 UTC
status string Nu scraped → tested → published → verified
vdate date Nu Data verificării, setată când statusul devine verified
expirationdate date Nu Data estimată de expirare = vdate + 30 zile maxim

Exemplu: Software Engineer la Holcim România

Câmp Valoare
url https://www.holcim.ro/jobs/software-engineer-123
title Software Engineer
company HOLCIM ROMÂNIA S.A.
cif 361324
location București, Cluj-Napoca
tags javascript, react, node.js, 3-5 ani experienta
workmode hybrid
salary 7000-10000 RON
date 2026-06-26T10:00:00Z
status verified
vdate 2026-06-26T12:00:00Z
expirationdate 2026-07-26

Surse de date

Anunțurile de locuri de muncă provin din scrapere create de voluntari:

1. Scrapere automate

Fiecare companie parteneră are un scraper dedicat care extrage anunțurile de pe site-ul oficial de cariere. Scraper-ele sunt scrise de voluntari și rulează periodic prin GitHub Actions.

  • Format: fișier .md cu configurația scraperului
  • Date colectate: url, title, company, location, tags, workmode, salary
  • Frecvență: configurabilă per scraper (zilnic/săptămânal)

2. Validare automată

După colectare, datele trec printr-un proces de validare care verifică corectitudinea fiecărui câmp conform regulilor modelului.

3. Verificare manuală

Un voluntar verifică periodic anunțurile publicate pentru a confirma că sunt încă valide și corecte.

Ciclul de viață

Ciclul de viață al unui loc de muncă în sistem:

  1. Scraped — Scraperul extrage anunțul de pe site-ul companiei și îl adaugă în SOLR cu datele brute.
  2. Tested — Anunțul trece de validările automate: format URL, lungime titlu, enum-uri corecte etc.
  3. Published — Anunțul validat este publicat și devine vizibil pe peviitor.ro.
  4. Verified — Un voluntar confirmă manual că anunțul este încă valabil. Data verificării (vdate) este setată.
  5. Expirat — După 30 de zile de la verificare, anunțul expiră și nu mai este afișat, cu excepția cazului în care este re-verificat.

Feature-uri derivate

Feature-urile derivate din acest epic, ce vor fi detaliate separat:

  • Job Validation — reguli de validare pentru fiecare câmp al modelului
  • Job Search — motor de căutare full-text pe anunțuri
  • Job Expiration — gestionarea expirării automate a anunțurilor
  • Duplicate Detection — detectarea anunțurilor duplicate
  • Bulk Import — import în masă pentru companii cu multe poziții
  • Company Matching — asocierea automată a anunțului cu compania corectă
