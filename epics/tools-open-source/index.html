TOOLS OPEN SOURCE
Tot ce folosim este open source. De la GitHub la Apache SOLR, de la Python la Docker, fiecare unealtă din acest proiect este liberă, transparentă și construită de comunitate.

Principiu

peviitor.ro nu doar că produce open source — și consumă open source. Fiecare tehnologie din stack este aleasă pentru că este liberă, bine întreținută și accesibilă oricui.

Un voluntar poate contribui fără să cumpere nimic, fără licențe plătite, fără platforme proprietare. Un laptop și o conexiune la internet sunt suficiente.

GitHub

GitHub este centrul proiectului. Tot codul, toate discuțiile, toate sarcinile și tot deployment-ul trec prin el:

  • Repository-uri — fiecare componentă are propriul repo: scrapere, API, front-end, configurare, documentație
  • GitHub Actions — CI/CD: rulează scrapere, testează validări, publică pe GitHub Pages
  • Issues — bug-uri, feature-uri, sugestii și sarcini
  • Projects — board Kanban pentru management vizual
  • Pull Requests — review de cod, discuții și aprobare
  • GitHub Pages — găzduire cerințe și documentație
  • Discussions — dezbateri și anunțuri

Apache SOLR

Motorul de căutare. Indexează sute de mii de documente și răspunde în milisecunde. Configurabil via Schema API, extensibil cu plugin-uri, scalabil pe cluster. (Detalii în EPIC-006.)

Python

Limbajul principal pentru scraping. Ecosistemul PyPI oferă biblioteci pentru orice nevoie: Requests, BeautifulSoup, Scrapy, Selenium, Playwright, FastAPI (pentru API), pytest (pentru testare).

Docker

SOLR rulează în container Docker. Un simplu docker-compose up pornește întregul mediu de căutare local. Volumele persistente asigură că datele nu se pierd la restart.

Discord

Comunitatea se adună pe Discord. Aici se discută scrapere noi, se raportează bug-uri, se cer help și se celebrează realizări. Notificările automate din GitHub Actions ajung tot aici.

De ce open source

  • Zero barieră financiară — oricine poate contribui fără să plătească
  • Transparență — codul este public, oricine îl poate verifica, modifica și îmbunătăți
  • Securitate — vulnerabilitățile sunt descoperite și reparate de comunitate
  • Durabilitate — nu depindem de un vendor sau de o licență care poate dispărea
  • Educație — oricine poate învăța citind codul real al unui proiect în producție

Feature-uri derivate

  • Development Setup Guide — ghid de configurare a mediului local
  • Docker Compose Stack — configurația Docker pentru SOLR + API
  • GitHub Actions Templates — workflow-uri standard pentru scrapere
  • VS Code Workspace — configurare recomandată pentru editor
  • Contributor Onboarding — ghid pentru noii voluntari
  • Tooling Scripts — scripturi de automatizare a sarcinilor comune
